# リファクタリングスキル

## 概要

既存コードの外部動作を変えずに、内部構造を改善するスキルです。

## リファクタリング手法

### 1. 関数の抽出 (Extract Function)
長い関数を意味のある単位に分割します。

**判定基準:**
- 関数が20行を超える
- コメントで区切られたブロックがある
- 同じインデントレベルに異なる処理が混在

### 2. 重複コードの排除 (DRY)
類似コードを共通化します。

**判定基準:**
- 3行以上の同一コードが2箇所以上に存在
- パターンが類似（パラメータのみ異なる）

### 3. 命名の改善 (Rename)
変数名・関数名をより明確にします。

**命名ルール:**
- 変数: 名詞（何を保持しているか）
- 関数: 動詞+名詞（何をするか）
- bool: is/has/can + 形容詞/名詞
- 配列: 複数形

### 4. デザインパターンの適用
適切なデザインパターンを提案・適用します。

### 5. 型の強化
any型の排除、Union型の活用、Genericsの適用。

## ワークフロー

1. **分析**: 対象コードの問題点を特定
2. **計画**: リファクタリング手順を提示
3. **確認**: ユーザーの承認を得る
4. **実行**: 段階的にリファクタリングを実施
5. **検証**: テストが通ることを確認

## 使用例

```
/refactoring src/services/userService.ts
```
