# コードレビュースキル

## 概要

コードの品質を多角的に評価し、改善提案を行うスキルです。

## レビュー観点

以下の観点からコードをレビューします：

### 1. 🐛 バグ・エラー
- ロジックエラー
- エッジケースの考慮漏れ
- null/undefined チェック
- 型の不整合

### 2. 🔒 セキュリティ
- SQLインジェクション
- XSS（クロスサイトスクリプティング）
- 認証・認可の不備
- 機密情報のハードコーディング
- 入力バリデーション不足

### 3. ⚡ パフォーマンス
- N+1 クエリ
- 不要なリレンダリング
- メモリリーク
- 非効率なアルゴリズム

### 4. 📖 可読性・保守性
- 命名規則の統一
- 関数の単一責任原則
- コメント・ドキュメントの充実度
- DRY 原則の遵守

### 5. ✅ テスト
- テストカバレッジ
- エッジケースのテスト
- テストの可読性

## 出力フォーマット

```markdown
## コードレビュー結果

### 🔴 重大（修正必須）
- [ファイル:行] 問題の説明
  - 推奨修正:  コード例

### 🟡 改善推奨
- [ファイル:行] 問題の説明
  - 推奨修正: コード例

### 🟢 良い点
- 良いプラクティスの記述

### 📊 サマリー
- 重大: X件 / 改善推奨: Y件 / 良い点: Z件
```

## 使用例

```
/code-review src/api/users.ts
```

または、PRの差分をレビュー：

```
/code-review !git diff main...HEAD
```
