# デバッグスキル

## 概要

バグの原因を体系的に調査し、修正を支援するスキルです。

## デバッグワークフロー

### 1. 情報収集
- エラーメッセージの確認
- スタックトレースの解析
- 再現手順の特定
- 関連するログの確認

### 2. 仮説立案
- エラーの種類に基づく原因候補のリストアップ
- 直近の変更箇所の確認
- 影響範囲の特定

### 3. 検証
- デバッグログの追加
- 変数の状態確認
- 条件分岐の追跡
- 最小再現ケースの作成

### 4. 修正
- 根本原因の修正
- リグレッションテストの追加
- 関連する問題の確認

### 5. ドキュメント
- 原因と修正内容の記録
- 再発防止策の提案

## よくあるエラーパターン

| エラータイプ | チェックポイント |
|---|---|
| TypeError | null/undefined チェック、型変換 |
| ReferenceError | 変数のスコープ、import文 |
| NetworkError | API URL、CORS、認証トークン |
| SyntaxError | JSON パース、テンプレートリテラル |
| TimeoutError | 非同期処理、Promise チェーン |

## 出力フォーマット

```markdown
## デバッグレポート

### 🔍 症状
- 報告された問題の要約

### 🎯 根本原因
- 原因の詳細な説明

### 🔧 修正内容
- 変更したファイルと内容

### ✅ 検証結果
- 修正後の動作確認結果

### 🛡️ 再発防止策
- 今後同様の問題を防ぐための提案
```

## 使用例

```
/debugging "TypeError: Cannot read property 'map' of undefined at UserList.tsx:45"
```
